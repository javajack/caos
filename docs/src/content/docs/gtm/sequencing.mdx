## The ten hard rules

These exist to prevent the most common failure modes and are non-negotiable.

<Aside type="danger" title="Hard rules — non-negotiable">
1. **CA wedge first**. Until 1,000 paying CA firms or ₹3 cr ARR (whichever earlier), no more than 20% of engineering capacity on enterprise-only features. All shared primitives count as CA features.
2. **Never replace the ERP**. Always plug into Tally / SAP / Oracle / Zoho / NetSuite / BUSY. Position as "the messy edges." Build first-class connectors.
3. **Don't sell direct to a CA's existing client without the CA knowing**. Build a CA-referral compensation engine (15-20% rev share). Make CAs feel they own the customer relationship.
4. **Don't sell direct to an audit firm's corporate client without the audit firm knowing**. Same principle as #3, applied to the audit channel.
5. **Don't dual-brand prematurely**. Single brand (CAOS) until ₹500 cr ARR.
6. **Don't promise on-prem unless single deal > ₹2 cr/year**. Pre-build VPC-isolated tenant for BFSI / PSU clients instead.
7. **Don't accept enterprise RFPs that require &lt;60-day procurement turnaround**. Walk away from procurement processes designed around incumbents.
8. **Pricing must be transparent**. Publish all SKUs and add-on prices on the website. No "enterprise: contact sales" black box — Indian buyers hate it.
9. **DPI participation is a regulatory commitment, not a feature**. Once registered as FIU / ERI / GSP partner, the obligations are real (data audit trails, consent UX, breach notification). Don't take registrations lightly.
10. **No vanity metrics**. Track ARR + active paying logos + cohort retention + flywheel-attributed acquisitions. Skip "trial signups," "page views," "social mentions" in board packs.
</Aside>

## What kills the sequence

Three failure modes that break the 60-month plan and require strategic reset, not just adjustment:

<Aside type="caution" title="Sequence-breakers">
- **CA wedge fails to ignite by Month 12**. If &lt;500 CA firms onboarded by M12, Phase 2 vendor network has no mass to build on. Mitigation: aggressive ICAI MOU + branch outreach + 5K free starter seats for new CAs ([file 12 lever](/caos/method/data-atlas/)). Fallback: pivot to direct-to-business if CA channel fails — but ARPU is 10x harder to justify without CA's cost-of-acquisition.
- **Audit firm channel rejection in Year 2**. If 0 of 3-5 mid-tier firms convert from pilot to per-engagement license by M24, Phase 3 doesn't start. Mitigation: position CAOS as the *client-facing* layer that integrates *into* their existing tools (Caseware, internal IP), not as a replacement. Lighthouse with 1 mid-tier first (Grant Thornton or BDO) before approaching others.
- **Zoho ships Practice + Books + Compliance bundle free for 3 years**. This is the [INSIGHT 7 risk from file 08](/caos/competitors/enterprise-threats/) applied to the full flywheel. Mitigation: race to lock in 100+ enterprise + 5 audit firms before Zoho's Indian compliance pack ships. Audit Workspace + DPI graph access + vendor network are all things Zoho doesn't currently have.
</Aside>

For the full failure-mode + counterfactual analysis, see [Five failure modes](/caos/risks/failure-modes/) and [Counterfactual scenarios](/caos/risks/counterfactual/).

## What sequencing means for hiring

The flywheel sequencing dictates the hiring sequence. The biggest mistake CAOS could make is hiring an enterprise sales leader in Year 1 — they'd have nothing to sell against (no Audit Workspace, no SOC 2, no enterprise SKU maturity). They'd burn ₹40-80 lakh/year and grind for 12 months before producing one logo.

The right hiring order:

| Phase | Roles to hire |
|---|---|
| Phase 0 (M1-M6) | 2 founding engineers; 1 product designer; 1 CA-network operator (ex-CA, well-connected at branch level) |
| Phase 1 (M7-M12) | +6 engineers; +1 PM; +1 partner-channel manager (ex-Tally / ex-ClearTax channel); +2 CA-success managers |
| Phase 2 (M13-M24) | +10 engineers; +2 PMs; +1 mid-tier-audit-firm partnership lead; +3 SDRs for direct mid-market; +1 DPI / regulatory ops manager |
| Phase 3 (M25-M36) | +15 engineers; +1 enterprise sales leader; +3 enterprise AEs; +1 SOC 2 / security lead; +1 customer success leader; +1 lender-partnership manager |
| Phase 4 (M37-M60) | +30+ engineers; +scale enterprise sales motion; +international expansion (foreign sub support per [Decision D6](/caos/decisions/d1-d7/)) |

## Hiring the channel partner ops layer

The single most underestimated hire in the plan is the **partner-channel operations manager** at M9-M12 (Phase 1 transition). This person owns deal registration, partner certification, MDF allocation, partner portal, quarterly business reviews. Without them, the Tally Master pilot stalls because partners can't track commissions and pipeline. The role typically pays ₹25-40 lakh/year for someone with 5+ years at Tally / ClearTax / Zoho channel teams.

## Connection to architecture

The sequencing is only credible if the [10+8 architectural pre-investments](/caos/decisions/architectural-pre-investments/) ship in Phase 0-3. Specifically:

- **Phase 0 must ship**: Person decoupled, entity as canonical noun, currency column reserved, doctype namespace reservations, public REST API contract drafted, webhook infrastructure
- **Phase 1 must ship**: Notice intake doctype + manual triage, multi-entity grouping primitive, vendor master schema
- **Phase 2 must ship**: AA TSP/FIU integration architecture, eMudhra/NSDL DSC integration, MSME 45-day watchlist
- **Phase 3 must ship**: NFRA-compliant audit working-paper schema, public-graph query API with consent-flow
- **Phase 4 must ship**: ESG/BRSR Core taxonomy module, DEDICATED tenant model

If any of these slip, the corresponding phase's GTM motion stalls. This is why the architectural pre-investments are no-regret and Phase-0-locked.
