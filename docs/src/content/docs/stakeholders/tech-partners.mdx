The Indian financial-document fabric does not exist in a vacuum. It runs on top of a **dense layer of tech and service partners** — accounting software incumbents, GSP/IRP/ERI government-portal aggregators, certificate authorities, eSign providers, and corporate payment platforms. Most of these are **integration partners** for CAOS (the wedge is "layer above Tally + plug into ERPs + connect to GSPs"); a few are **competitive context** in adjacent segments.

This page maps the tech-partner ecosystem with verified populations and revenue, organised by the role each plays in the [composite flywheel](/caos/flywheel/composite/) substrate.

<Aside type="tip" title="Why tech partners are stakeholder S13">
elevates the tech-vendor-partner pool to S13 because a CAOS entity that does not speak Tally / GSTN-via-GSP / DSC-via-CA / eSign-via-Signzy-or-Leegality / payment-via-Razorpay-or-Cashfree is functionally inoperative in Indian compliance. They are obligatory integration counterparties — and their commercial roadmaps shape what CAOS can ship.
</Aside>

## GSPs — GST Suvidha Providers

GSPs are GSTN-empanelled technology providers that offer authenticated APIs into the GST system. **62 GSPs are empanelled across the 5 batches** ([Microvistatech](https://www.microvistatech.com/blog/gst-suvidha-providers-gsp-list-2025/)). The notable first-mover for Invoice Management System (IMS):

| Provider | Notable status | Source |
|---|---|---|
| **Cygnet.One** | First ASP-GSP live with IMS (Invoice Management System) under GST regime | [CXOToday Oct 2024](https://cxotoday.com/press-release/cygnet-one-the-first-asp-gsp-platform-to-go-live-with-invoice-management-system-under-the-gst-regime/) |
| **62 total GSPs** | Across 5 GSTN empanelment batches | [Microvistatech](https://www.microvistatech.com/blog/gst-suvidha-providers-gsp-list-2025/) |

The CAOS strategy here is to be **ASP-first** (Application Service Provider — riding on top of an existing GSP's authenticated APIs) rather than pursuing direct GSP licensing. ASP-under-existing-GSP ships in 6-9 months; direct GSP licensing requires 12-18 months plus infrastructure investment).

## ERIs — e-Return Intermediaries

ERIs are CBDT-authorised entities for filing income tax returns on behalf of taxpayers. There are **~50+ named ERIs** (Type-1 and Type-2 split not publicly disclosed) ([ClearTax — ERI Income Tax](https://cleartax.in/s/e-return-intermediary-eri-income-tax)).

Type 1 ERIs can use the income tax department's free utility to file returns. Type 2 ERIs can develop their own software using the ERI scheme APIs — a richer integration that allows automated bulk filing.

| Metric | Value | Source |
|---|---|---|
| ERIs (named entities) | **"50+"** (Type-1 / Type-2 split not publicly disclosed) | [ClearTax — ERI Income Tax](https://cleartax.in/s/e-return-intermediary-eri-income-tax) |

CAOS's [decision D7](/caos/decisions/d7-aa-gsp-eri-participation/) is to pursue Type 2 ERI status in Y3, giving CAOS authenticated bulk-ITR filing capability.

These platforms serve overlapping but distinct buyers:

- **Razorpay / Cashfree** — payment gateway + payroll + corporate payouts
- **Perfios / M2P / Decentro** — fintech-infra plumbing for lender + neobank stack
- **EnKash / Volopay / Happay** — corporate card + AP automation (closest competitive context for CAOS AP modules)

CAOS integrates with these as payment-execution layers (NEFT / RTGS file generation, vendor payment trigger) but does not compete on payment processing. The wedge is the **AP intelligence above payment** (vendor onboarding, MSME 45-day, GSTR-2B reco) — none of which the payment infra players own at depth.
